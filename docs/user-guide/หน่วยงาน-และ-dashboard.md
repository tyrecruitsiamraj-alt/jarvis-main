# คู่มือการใช้งาน Jarvis — หน้าหน่วยงาน และ Dashboard

คู่มือสำหรับผู้ใช้งานจริง (สรรหา / คัดสรร / OPL / Supervisor / Admin)  
โฟกัส 2 หน้าหลักที่ใช้ประจำ: **หน่วยงาน** และ **Dashboard (Request Control Tower)**

---

## 1. เริ่มต้นใช้งาน

### เข้าแอป
1. เปิดเว็บ Jarvis แล้วเข้าสู่ระบบด้วยบัญชีที่ได้รับ
2. หลังล็อกอินจะเห็นเมนูหลัก (Home) หรือ Role Hub ตามสิทธิ์

### ไปหน้าไหนได้จากไหน

| อยากไป | ทางเข้า |
|--------|--------|
| **หน่วยงาน** (รายการใบขอ) | Bottom nav → **หน่วยงาน** · หรือการ์ด **หน่วยงาน** บนหน้า Home |
| **ภาพรวมหน่วยงาน** | Role Hub (Supervisor+) → **แดชบอร์ดหน่วยงาน** |
| **Dashboard** | Bottom nav → **Dashboard** · หรือการ์ด **Dashboard** บนหน้า Home |

> แอปจะจำว่าคุณอยู่หน้า list หรือ overview ของหน่วยงานล่าสุด และจำตัวกรอง/หน้าที่เปิดอยู่ — กดปุ่มหน่วยงานอีกครั้งจะกลับมาที่เดิม

---

## 2. หน้าหน่วยงาน

มี 2 มุมมองหลัก:

| มุมมอง | ที่อยู่โดยประมาณ | ใช้เมื่อ |
|--------|------------------|----------|
| **รายการงาน** | `/jobs/list` | หาใบขอ, กรอง, ดูสถานะวันต่อวัน |
| **ภาพรวม** | `/jobs/overview` | ดูสรุปจำนวนงานทั้งหมด / ดำเนินการ / ปิดแล้ว |

### 2.1 ภาพรวมหน่วยงาน (`/jobs/overview`)

**เห็นอะไร**
- การ์ดสรุป: **งานทั้งหมด** · **ดำเนินการ** · **ปิดแล้ว** (นับเป็นตำแหน่ง + จำนวนใบขอ)
- ตัวกรองพื้นฐาน: แผนก, ลักษณะงานย่อย, ผู้รับผิดชอบ, สถานะใบขอ
- รายการ **งานล่าสุด** (ประมาณ 4 ใบ)
- ปุ่ม **รายการงานทั้งหมด** → ไปหน้ารายการเต็ม

**วิธีใช้เร็ว**
1. กรองแผนก / ผู้รับผิดชอบก่อน
2. กดการ์ดสรุปเพื่อเปิดรายชื่อใบขอในกลุ่มนั้น
3. กดใบขอเพื่อเปิดรายละเอียด

---

### 2.2 รายการงานทั้งหมด (`/jobs/list`)

นี่คือหน้าที่ใช้ประจำในการติดตามใบขอ

#### ค้นหา
ช่องค้นหา: เลขที่ใบขอ, ชื่อหน่วยงาน, ผู้รับผิดชอบ ฯลฯ  
ถ้าพิมพ์เลขที่ใบขอแบบ Siamraj ระบบจะพยายามหาใบนั้นให้ด้วย

#### แท็บสถานะใบงาน
- **ทั้งหมด**
- **ดำเนินการ** — ยังไม่ปิด
- **ปิดแล้ว**

#### ตัวกรองสำคัญ

| ตัวกรอง | ความหมายสั้นๆ |
|---------|----------------|
| **แผนก** | แผนกที่ดูแลใบขอ (Admin เห็นทุกแผนก · คนอื่นมักถูกล็อกตามแผนกตัวเอง) |
| **ลักษณะงานย่อย** | ประเภทงานย่อยในใบขอ |
| **ปี (พ.ศ.)** | ปีของใบขอ |
| **เจ้าหน้าที่สรรหา / คัดสรร / OPL** | กรองตามผู้รับผิดชอบ · มีตัวเลือก **ยังไม่ถูก Assign** |
| **สถานะใบขอ** | ฉุกเฉิน/ย้อนหลัง · ฉุกเฉิน · ล่วงหน้า |
| **สถานะทำงาน** | สถานะที่ทีมอัปเดตใน Jarvis (ดูตารางด้านล่าง) |
| **หมายเหตุ** | มีหมายเหตุ / ไม่มีหมายเหตุ |
| **ส่งคนแทน** | ส่งคนแทน / ไม่ส่ง / ยังไม่ระบุ |
| **วันผ่านมา** | อายุงานนับจากวันที่ต้องการคน |
| **เรียงลำดับ** | เรียงตามผ่านมา / วันที่กรอก / ผู้รับผิดชอบ |

#### ตัวเลือก: เปิดใบขอในแท็บใหม่
ติ๊ก **เปิดใบขอในแท็บใหม่** หรือใช้ Ctrl/⌘ + คลิก (หรือคลิกปุ่มกลางเมาส์)  
เหมาะตอนเทียบหลายใบพร้อมกัน

#### คอลัมน์ตาราง (คอมพิวเตอร์)

ลำดับคอลัมน์:

1. เลขที่ใบขอ  
2. ผ่านมา  
3. หน่วยงาน  
4. วันที่กรอก  
5. วันที่ต้องการ  
6. คงเหลือ  
7. ประเภทใบขอ  
8. ตำแหน่ง  
9. ลักษณะงานย่อย  
10. ผู้ลาออก  
11. ผู้รับผิดชอบ  
12. **ส่งคนแทน**  
13. **สถานะทำงาน**  
14. หมายเหตุ  

บนมือถือจะแสดงเป็นการ์ดแทนตาราง แต่ข้อมูลหลักเหมือนกัน

### 2.3 ความหมายของสถานะ / แบดจ์

#### สถานะใบขอ (ความเร่งด่วน)

| สถานะ | ความหมาย |
|-------|----------|
| **ฉุกเฉิน/ย้อนหลัง** | วันที่ต้องการคน **ก่อน** วันที่กรอกใบขอ |
| **ฉุกเฉิน** | จากวันกรอกถึงวันต้องการ **น้อยกว่า 7 วัน** |
| **ล่วงหน้า** | จากวันกรอกถึงวันต้องการ **ตั้งแต่ 7 วันขึ้นไป** |

#### สถานะทำงาน (อัปเดตใน Jarvis)

| สถานะ | ความหมายโดยทั่วไป |
|-------|-------------------|
| **ดำเนินการ** | กำลังสรรหาอยู่ (ค่าเริ่มต้น) |
| **เริ่มประเมิน** | มีคนแล้ว เริ่มประเมิน |
| **รอแจ้งเข้า** | รอขั้นตอนแจ้งเข้า |
| **รอสัมภาษณ์** | นัด/รอสัมภาษณ์ |
| **รอผลสัมภาษณ์** | สัมภาษณ์แล้ว รอผล |
| **รอเริ่มงาน** | ได้คนแล้ว รอวันเริ่มงาน |
| **งานรายวัน** | จ้างแบบรายวัน |
| **จ่ายรายวัน** | จ่ายค่าแรงรายวัน |

> สถานะนี้เป็นของ Jarvis ไม่ได้แก้ข้อมูลต้นทาง Siamraj

#### ส่งคนแทน
- **ส่งคนแทน** / **ไม่ส่งคนแทน** / **—** (ยังไม่ระบุ)

#### คงเหลือ
จำนวนตำแหน่งที่ยังต้องหาในใบนั้น

#### ผ่านมา
อายุงานนับจากวันที่ต้องการคน เช่น `ล่วงหน้า` หรือ `N วัน`

---

### 2.4 เปิดรายละเอียดใบขอ (Siamraj)

คลิกแถว/การ์ด → ไปหน้ารายละเอียดใบขอ

**ส่วนที่ดูได้อย่างเดียว (จาก Siamraj)**
- เลขที่ใบขอ, หน่วยงาน, สถานที่
- วันที่กรอก / วันที่ต้องการ
- ขอมา · หาได้แล้ว · คงเหลือ
- ประเภทใบขอ, ตำแหน่ง, ผู้ลาออก, รายได้ ฯลฯ

**ส่วนที่แก้ใน Jarvis ได้ (ตามสิทธิ์)**

| ส่วน | ใครแก้ได้โดยทั่วไป |
|------|---------------------|
| ผู้รับผิดชอบ (สรรหา / คัดสรร / OPL) | Supervisor ขึ้นไป |
| หมายเหตุ | Staff ขึ้นไป (มีสิทธิ์แก้ไขหมายเหตุ) |
| ส่งคนแทน | Staff ขึ้นไป (มีสิทธิ์แก้ไขหมายเหตุ) |
| สถานะทำงาน (+ ชื่อคน / วันที่) | Staff ขึ้นไป (มีสิทธิ์แก้ไขหมายเหตุ) |

> OPL โดยปกติ **อ่าน** ได้ แต่ไม่แก้หมายเหตุ/สถานะทำงาน  
> ข้อมูลต้นทาง Siamraj ต้องแก้ที่ระบบต้นทาง

#### ปุ่มย้อนกลับ
กดกลับจะพาไปหน้ารายการเดิม (พร้อมตัวกรอง/หน้าที่เคยเปิด) ไม่หลุดกลับหน้าแรก

---

### 2.5 เคล็ดลับใช้งานหน่วยงาน

1. **เริ่มจากกรองแผนก + ผู้รับผิดชอบ + สถานะทำงาน** ก่อนค้นหาทีละใบ  
2. ใช้ **ยังไม่ถูก Assign** เพื่อหาใบที่ยังไม่มีเจ้าของ  
3. ใช้ **วันผ่านมา 30 วันขึ้นไป** หาคิวงานค้างเก่า  
4. อัปเดต **สถานะทำงาน** และ **หมายเหตุ** บนหน้า detail ให้ทีมตามงานได้โดยไม่ต้องเปิดระบบต้นทาง  
5. ถ้าเปิดหลายใบพร้อมกัน ให้ติ๊ก **เปิดในแท็บใหม่**

## 3. หน้า Dashboard (Request Control Tower)

เส้นทาง: `/dashboard`  
หัวข้อบนหน้า: **Request Control Tower**

ใช้ดูภาพรวมงานสรรหาเป็นช่วงเวลา — จำนวนอัตราที่เข้ามา / ปิดแล้ว / ยกเลิก / คงเหลือ และงานที่ต้องติดตาม

### 3.1 ใครใช้หน้านี้
- Staff / Supervisor / Admin (ตามสิทธิ์เมนู `dashboard`)
- คนที่ไม่ใช่ Admin มักเห็นเฉพาะแผนกของตัวเอง

### 3.2 ตัวกรองหลัก

| ตัวกรอง | ใช้ทำอะไร |
|---------|-----------|
| **ช่วงเวลา** | ทั้งหมด · เดือนนี้ · เดือนก่อน · เลือกเดือน/ปี · ช่วงวันที่ |
| **แผนก / ลักษณะงานย่อย / หน่วยงาน** | โฟกัสขอบเขตงาน |
| **สรรหา / คัดสรร / OPL** | ดูภาระงานรายบุคคล |
| **สถานะใบขอ** | โฟกัสงานฉุกเฉินหรือล่วงหน้า |
| **ค้นหา** | ค้นใบงาน / คน / ปลายทาง ในตารางติดตาม |

มีปุ่ม **รีเฟรช** และ **Export CSV**

---

### 3.3 สรุปอัตราในช่วงที่เลือก

การ์ดหลัก 4 ใบ:

| การ์ด | ความหมายสั้นๆ |
|-------|----------------|
| **เข้ามา** | อัตรา/ตำแหน่งที่ขอในช่วงที่เลือก |
| **ปิดแล้ว** | อัตราที่หาได้แล้ว (description ใน UI: หาได้แล้ว) |
| **ยกเลิก** | อัตราที่ยกเลิก |
| **คงเหลือ** | อัตราที่ยังต้องหา |

แนวคิดหลัก:

```text
เข้ามา − ปิดแล้ว − ยกเลิก ≈ คงเหลือ
```

> **ปิดแล้ว** ใน Dashboard หมายถึงยอดที่ **หาได้แล้ว (ตำแหน่ง)**  
> ไม่ใช่คำว่า “ปิดครบใบขอ” (ใบปิดครบทุกตำแหน่ง) — อย่าสับสนสองคำนี้

---

### 3.4 ส่วนอื่นๆ บน Dashboard

1. **สถานะทำงาน (นับอัตรา)** — กระจายตามสถานะที่ทีมอัปเดตใน Jarvis  
2. **วันผ่านมา** — ล่วงหน้า / 1–7 / 8–15 / 16–30 / 30+ วัน  
3. **Life Cycle ตามประเภทใบขอ** — เข้ามา / ปิดแล้ว / ยกเลิก / คงเหลือ แยกประเภท เช่น ลาออก · เปลี่ยนตัว · เพิ่มอัตรา · เปิดไซต์ · อื่นๆ  
4. **แนวโน้มรายเดือน** — กราฟเข้ามา / ปิดแล้ว / ยกเลิก / คงเหลือ  
5. **ประเภทใบขอรายเดือน** — แตกยอดเข้ามาตามประเภท  
6. **ภาระงานตามผู้รับผิดชอบ** — มี / ปิด / คงเหลือ ต่อคน  
7. **ภาระงานตามหน่วยงาน** — คงเหลือรายหน่วยงาน  
8. **งานที่ต้องติดตาม** — ตารางใบงานที่ต้องโฟกัส

#### คอลัมน์ตาราง “งานที่ต้องติดตาม” (สำคัญ)
- ใบงาน  
- **ขอมา**  
- **หาได้แล้ว**  
- **ยกเลิก**  
- **เหลือหา**  
- ประเภท · สถานะ · สถานะทำงาน · SLA · ฯลฯ  

คลิกใบงานเพื่อเปิดรายละเอียด (แล้วกลับมา Dashboard ได้)

---

### 3.5 คำศัพท์ที่ควรจำบน Dashboard

| คำในหน้า | ความหมายใช้งาน |
|----------|----------------|
| **เข้ามา** | ยอดที่ขอในช่วง |
| **ปิดแล้ว / หาได้แล้ว** | ยอดตำแหน่งที่หาคนได้แล้ว |
| **ยกเลิก** | ยอดที่ยกเลิก |
| **คงเหลือ / เหลือหา** | ยอดที่ยังต้องหา |
| **ขอมา** | จำนวนที่ขอในใบนั้น |
| **งานค้าง** | งานเก่าที่ยังค้างอยู่ในระบบ |

---

### 3.6 เคล็ดลับใช้งาน Dashboard

1. ตั้ง **ช่วงเวลา** ให้ชัดก่อนอ่านตัวเลข (เดือนนี้ vs เดือนก่อน)  
2. ดูการ์ด **คงเหลือ** ก่อน แล้วค่อยเจาะประเภท/ผู้รับผิดชอบ  
3. เปิด **งานที่ต้องติดตาม** เมื่อจะลงมือทำงานรายใบ  
4. ใช้ Export CSV เมื่อต้องส่งรายงานหรือวิเคราะห์นอกระบบ  
5. ถ้าตัวเลขดูไม่ลงตัว ให้เช็กว่ากำลังดู **ตำแหน่ง (อัตรา)** ไม่ใช่ **จำนวนใบขอ**

## 4. หน่วยงาน vs Dashboard — ใช้เมื่อไหน

| สถานการณ์ | แนะนำใช้ |
|-----------|----------|
| หาใบขอวันนี้ อัปเดตสถานะ/หมายเหตุ | **หน่วยงาน → รายการ** |
| มอบหมายผู้รับผิดชอบ | **หน่วยงาน → รายละเอียดใบขอ** |
| ดูว่าแผนก/คนไหนค้างเยอะ | **Dashboard** |
| รายงานรายเดือน / แนวโน้ม | **Dashboard** |
| โฟกัสงานฉุกเฉินรายใบ | **หน่วยงาน** (กรองสถานะใบขอ) หรือตารางติดตามบน **Dashboard** |

สรุปสั้นๆ:
- **หน่วยงาน** = ทำงานรายใบ  
- **Dashboard** = ดูภาพรวมและวางแผน

ตอบ: ไม่ควรหาย — ระบบจำ URL/ตัวกรองของหน้ารายการไว้ ถ้าหายให้รีเฟรชแล้วลองเปิดจากปุ่มหน่วยงานอีกครั้ง

**ถาม: แก้รายได้/ตำแหน่งในใบขอ Siamraj จาก Jarvis ได้ไหม?**  
ตอบ: ไม่ได้ — ส่วนนั้นอ่านอย่างเดียว ต้องแก้ที่ระบบต้นทาง

**ถาม: “ปิดแล้ว” บน Dashboard กับ “ปิดครบใบขอ” ต่างกันไหม?**  
ตอบ: ต่างกัน  
- **ปิดแล้ว / หาได้แล้ว** = นับตำแหน่งที่หาคนได้  
- **ปิดครบใบขอ** = ใบนั้นหาครบทุกตำแหน่งแล้ว (คำนี้ไม่ได้โชว์เป็นหัวข้อหลักบน Dashboard)

**ถาม: สถานะทำงานกับสถานะใบขอต่างกันอย่างไร?**  
ตอบ:  
- **สถานะใบขอ** = ความเร่งด่วนจากวันที่ต้องการคน  
- **สถานะทำงาน** = ขั้นที่ทีมอัปเดตเองใน Jarvis
